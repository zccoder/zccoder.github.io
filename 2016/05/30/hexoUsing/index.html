ubuntu平台下，使用hexo搭建github博客

基本步骤

写博文前的准备工作

  • 安装nodejs,hexo;
  • 启动hexo服务器;
  • hexo总结

（１）按照官网上的方法：Ubuntu下安装nodejs

（２）通过npm安装：hexosudo npm install hexo -g
注：-g是指全局安装hexo
（３）初始化一个blog 并且启动服务器：

此时文件目录为

blog
    _config_yml // 注配置文件
    db.json // 数据
    debug.log // 调试日志
    _node_mudules // nodejs 相关依赖
    package.json // 配置依赖
    scaffolds // 脚手架 - 也就是一个工具模板
    source // 存放blog正文的地方
    themes // 存放皮肤的地方

默认访问 http://localhost:4000

写博文

（１）新建文章(在blog文件夹下)
新建一篇主题为hello的blog：hexo new hello
（２）编辑 hello.md 就是编辑你的blog内容（hexo会在 source/_posts/ 下新建hello.md 文件）
（３）静态处理：因为我们的blog要部署到github静态服务器上面，所有我们还要将页面进行静态化hexo g（blog根目录下会生成public文件夹）
（４）修改blog目录下的 _config.yml 文件,打开 _config.yml 找到如下配置
deploy: type: git repo:https://github.com/leopardpan/leopardpan.github.io.git
注：hexo3.0以后的这里可能会报

title: zccoder
subtitle: 专注于Android、Java、移动互联网、项目管理、软件架构
description:
author: zccoder
language: zh-CN

hexo new”postName” #新建文章
hexo new page”pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本

hexo clean
hexo generate
hexo deploy

hexo deploy -g #生成加部署
hexo server -g #生成加预览
命令的简写为：
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
